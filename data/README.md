# data/

One directory per glider discipline, mirroring the layout of the external SSD:

```
data/
├── paragliders/     # parapente.ffvl.fr  (soaring-para, SOARING_PARA_DATA_ROOT)
└── hang_gliders/    # delta.ffvl.fr      (soaring-delta, SOARING_DELTA_DATA_ROOT)
```

Each holds two files, regenerated by `soaring-<para|delta> build-catalog`:

- `seasons_index.csv` — **versioned**. One row per season with links (list + XML export)
  and counts (total flights, with track, downloaded). It is the source of the thesis
  statistics (`scripts/reporting/generate_stats.py`).
- `catalog.csv` — **not versioned** (large derivative). One row per flight: metadata plus
  the local path of its track. Kept here only as a local copy for figure generation
  (`scripts/reporting/generate_preproc_figure.py`); refresh it from the SSD when needed.

The **raw data** (`.igc` files, archived XMLs, full catalog) is **not** stored in the repo:
it lives in `data_root` on the external SSD (see `configs/para_download.yaml` for paragliders
and `configs/delta_download.yaml` for hang gliders). Everything except the two
`seasons_index.csv` is excluded via `.gitignore`.
