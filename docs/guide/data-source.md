# Data source: the FFVL CFD

The data comes from the **Coupe Fédérale de Distance (CFD)** of the Fédération Française de
Vol Libre. For each season there is a flight list page; **the key point** is that each list
has a **complete XML export** that returns *all* flights in a single response.

## URLs, by season

For the season starting in year `Y` (e.g. `Y = 1999` → season 1999-2000):

| Resource | URL |
|---------|-----|
| List page (human-readable) | `https://parapente.ffvl.fr/cfd/liste/{Y}` |
| **XML export** (used by the downloader) | `https://parapente.ffvl.fr/cfd/liste/{Y}?xml=1` |

The full list of URLs and counts, season by season, is in
[`data/seasons_index.csv`](https://github.com/matteodisante/soaring-anomalous-transport)
(regenerated by `build-catalog`). The URL builders are in `soaring.acquisition.ffvl.seasons`
(see [API Reference](../reference.md)).

## What the XML contains

Each flight is a `<flight>` element with all metadata as attributes, **including the direct
link to the `.igc` file**. Real excerpt (season 1999-2000):

```xml
<flight id="20150770"
        flight_link="https://parapente.ffvl.fr/cfd/liste/vol/20150770"
        date="2000-00-00" pilot="ETIENNE GRASSART" flight_type="triangle"
        distance="35.59" takeOff="Les Ilettes" landing="Orbassy"
        igc_tracklog="53180"
        igc_tracklog_link="https://parapente.ffvl.fr/sites/.../…-53180.igc" .../>
```

Thanks to this, visiting individual flight pages is never necessary: everything is in the XML.
Parsing is handled by `soaring.acquisition.ffvl.catalog_xml`.

## Cloudflare

The site is protected by Cloudflare: a regular HTTP request receives `403` with a challenge
page. We use [`curl_cffi`](https://github.com/lexiforest/curl_cffi), which mimics the TLS
fingerprint of a real browser and bypasses the filter. The logic (with retry, backoff, and
session renewal) is in `soaring.acquisition.ffvl.http`.

!!! note "Courtesy"
    We download with a limited number of parallel requests and a jittered pause between
    requests, so as not to overload the FFVL servers. The data is public and used for
    academic research.
