# Data sources: the FFVL CFD

The data come from the **Coupe Fédérale de Distance (CFD)** of the Fédération Française de
Vol Libre (FFVL), which runs two parallel competitions on separate sites:

| Site | Glider type | CLI | Config |
|------|-------------|-----|--------|
| [parapente.ffvl.fr](https://parapente.ffvl.fr/cfd/liste) | Paragliders | `soaring-para` | `configs/para_download.yaml` |
| [delta.ffvl.fr](https://delta.ffvl.fr/cfd/liste) | Hang gliders (deltaplane) | `soaring-delta` | `configs/delta_download.yaml` |

Both sites have the same URL structure and XML export format; the same acquisition code
(`soaring.acquisition.ffvl`) handles them both.

## URLs, by season

For the season starting in year `Y` (e.g. `Y = 2014` → season 2014-2015):

| Resource | Paragliders | Hang gliders |
|---------|-------------|--------------|
| List page | `https://parapente.ffvl.fr/cfd/liste/{Y}` | `https://delta.ffvl.fr/cfd/liste/{Y}` |
| **XML export** | `https://parapente.ffvl.fr/cfd/liste/{Y}?xml=1` | `https://delta.ffvl.fr/cfd/liste/{Y}?xml=1` |

The full list of URLs and counts, season by season, is in `catalog.csv` / `seasons_index.csv`
(regenerated by `build-catalog`). The URL builders are in `soaring.acquisition.ffvl.seasons`
(see [API Reference](../reference.md)).

## Season ranges

| Source | First season | Last season |
|--------|-------------|-------------|
| Paragliders | 1999–2000 | 2025–2026 |
| Hang gliders | 2001–2002 | 2025–2026 |

## What the XML contains

Each flight is a `<flight>` element with all metadata as attributes, **including the direct
link to the `.igc` file**. Real excerpt:

```xml
<flight id="20150770"
        flight_link="https://parapente.ffvl.fr/cfd/liste/vol/20150770"
        date="2000-00-00" pilot="ETIENNE GRASSART" flight_type="triangle"
        distance="35.59" takeOff="Les Ilettes" landing="Orbassy"
        igc_tracklog="53180"
        igc_tracklog_link="https://parapente.ffvl.fr/sites/.../…-53180.igc" .../>
```

Thanks to this, visiting individual flight pages is never necessary: everything is in the XML.
Parsing is handled by `soaring.acquisition.ffvl.catalog_xml`.

## Cloudflare

Both sites are protected by Cloudflare: a regular HTTP request receives `403`. We use
[`curl_cffi`](https://github.com/lexiforest/curl_cffi), which mimics the TLS fingerprint of
a real browser and bypasses the filter. The logic (with retry, backoff, and session renewal)
is in `soaring.acquisition.ffvl.http`.

!!! note "Courtesy"
    We download with a limited number of parallel requests and a jittered pause between
    requests, so as not to overload the FFVL servers. The data is public and used for
    academic research.
